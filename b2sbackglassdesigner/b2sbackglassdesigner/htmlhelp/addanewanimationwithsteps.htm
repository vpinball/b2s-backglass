Add a new animation with steps

Creating a first B2S backglass ›› Managing animations ››

Let's add a sample animation at 'Close Encounters'.


Get into the 'Manage animations ...' window and click the button '... or add a new animation'. The focus should be in the 'Animation name' text box. Enter a name, here 'My test animation'. Enter some more values like you want to, pretty important is an interval in milliseconds. I have entered 200 ms.


Now look at the animation steps area please. Click the topmost button to the right of the animation steps list box. It's the button with the with sheet of paper and the yellow star. This button will add a new animation step at the end of the steps list. Here the step 1 is added.

Now click at one of the visible illumination frames at the designer's main window. The name of this frame is inserted into the 'Lamps On' and the 'Lamps Off' text box.

Last thing to do right now is to enter a 'Wait intervals after On' value in the according text box.


Here is what I have entered, I have clicked on the top left frame of the flying sacuer to get the name 'ST1' and entered the 'Wait intervals after On' value '1'.


Once again click on the 'Add a step' button and click another frame at the designer. Look at the 'step 2' entry in the steps list box. The 'Wait intervals after On' value is already inserted and got the same value as the last edited step.

Move on with some more steps.


At step 3 and especially at step 5 I have clicked a lot more frames like one. This is the result, 2 respectively 8 frames are inserted.


Of course there is no need to enter the same 'Lamps On' and 'Lamps Off' list at each step. Look at step 3, I have removed the frame 'ST2' in the according text box. This means, at step 3 the frame(s) with name 'ST2' are not switched off and stay on till step 5. After this step all 8 bulbs will be switched off.


And now test your new animation. Click the upper right button with the yellow bulb icon. At the first animation test all necessary bulb illumination is created. This will take a few seconds. After this time the animation preview mode will start.

Please notice: This animation preview mode is not this fast as the animations are running in the generated backglass. This is no problem with an interval of 200 milliseconds but an animation with a 20 ms interval is running too slow. This is a current technical problem in the designer.


And just one tip of the 'Some interesting tips' chapter: Illumination frames need a name to be involved into an animation. This name doesn't need to be unique. Name some frames with the same name to switch them on or off during an animation with only one name entry per step.
