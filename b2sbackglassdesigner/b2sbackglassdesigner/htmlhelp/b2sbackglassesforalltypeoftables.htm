B2S backglasses for all type of tables

What is the 'B2S Backglass Designer' able to create?


As I mentioned earlier, the 'B2S Backglass Designer' is able to create backglasses for EM tables, SS tables or original tables.

EM, Electro-mechanical, pinball machines were manufactured from the 1930's until the mid 1970's, when the transition to SS, solid state machines was made. Original tables aren't a replication of existing machines.


While SS tables are normally running 'Visual PinMAME' to emulate the CPUs and sound chips of the tables EMs and Originals are just controlled by the internal 'Visual Pinball' table script.

Because of that basic difference there are two modes of communication between the VP table and the B2S backglass:

  • 'VPinMAME ROM communication'
    Visual PinMAME is used to communicate between table and backglass. Normally the only necessary change in the VP table script is changing the 'Set Controller = CreateObject("VPinMAME.Controller")' to 'Set Controller = CreateObject("B2S.Server")'.
  • 'B2S data element communication'
    The VP table script is used to communicate between table and backglass.


Beyond that communication mode there are two totally different backglass output types:

  • 'B2S.Server 'directB2S' backglass'
    The output is a '*.directB2S' file. This file is a XML file with all necessary backglass info and needs the 'B2S.Server' to run. This output can be chosen for all types of tables and for all communication modes.
  • 'Microsoft Visual Studio 2010 project'
    The output is a Microsoft Visual Studio 2010 solution. This solution needs Visual Studio 2010 to be compiled. The final output is an backglass EXE file (like you know it already, this is the well-known 'Rosve' style). This output can be chosen for all types of tables but only for the VPinMAME ROM communication mode.
