Time to start with the LEDs.


I'm sure you know what an EM reel is. And of course you know a LED is a 'shiny sign'. But do you know some more background of backglass LEDs?  

LED displays are composed of several single LEDs. I.e. you can find 2 LED displays with up to 18 single LEDs or 6 or 7 LED displays with up to 7 LEDs each. A single LED is composed of several separated segments. You can find 7, 10, 14 or 16 segments each digit. Most of the early LED tables (Kiss, Lost World, Cosmic Gunfight, ...) are using 7-segmented LEDs, some early Gottlieb tables (Close Encounters, ...) are using 10-segmented LEDs, the newer tables (like Funhouse, Elvira, ...) are using 14-segmented LEDs. Very interesting, some pins have a mix of different LEDs, i.e. Taxi is using 14-segmented LEDs for the standard display but a 7-segmented LED for the taximeter. Main advantage of the LEDs with more segments is of course the ability to show much more different signs.

VPinMAME is sending the info which segments of each digit has to be lit.


Some infos about 'reels and LEDs' in the designer: There are two base types of designer 'reels or LEDs':

  • Image sets
    Image sets are a set of minimum 11 images. They are very limited since they have no possibility to access the segments individually (because each number is just an image). Right now there are predifined image sets for EM reels, EM credit reels and 7-segmented LEDs. A feature to import own image sets will be implemented sometime.
  • Rendered LEDs
    Rendered LEDs are rendered at runtime. There is nearly no limitation to these LEDs. Right now there are two types of LEDs, optically medium-level standard LEDs and high-level 'Dream 7' LEDs. Each of these two types can be selected in a 7-, 10- or 14-segmented version. 16-segmented will be implemented when necessary.


What does this mean for you?

  • First of all figure out which type of reels or LEDs you have to use. When you need ...
    • ... EM reels or EM credit reels, select (or import if possible) an EM reel or EM credit reel image set.
    • ... LEDs and you need ...
      • ... 10- or 14- (or even 16-) segmented LED select the rendered LEDs with matching segment count in either case.
      • ... 7-segmented LEDs select the rendered LEDs or a matching LED image set.
  • Now figure out how many digits the LED displays are having.
  • And your last job here is to figure out the order of your LED displays. Normally the LED displays are numbered logically (i.e. LED display of 'Player 1' is the first LED display, LED display of 'Player 2' is the second one, ...) but do not trust this too much.


Let's return to 'The Bally Game Show'. This is the second of two LED displays at the original backglass image from IPDB.ORG.


We do job 1 and 2 from our work list:  

  • Both LED displays are equal.
  • Each LED display has 16 digits.
  • It looks like each digit is made of 14 segments (this is not easy to identify, IMHO it's better discoverable at the non-light LED digits).


Back to the designer. We have to select the used type of LED. Click on 'Reels & LEDs' -> 'Choose Reel or LED type ...'


..., since we need 14-segmented LEDs we have to select the rendered LEDs. Check the 'Use 'Dream7' LEDs' check box, select the '14 segmented' previewed LED (there has to be a blue frame around the LED '14') and maybe have fun with the sliders of style and color. Then click 'Ok'.


These are the three score tool buttons:


Click the tool button in the middle, 'Add new reel or LED frame' and add a new score frame.

Since I zoomed to '125%' and scrolled the image down this is what I see


The 'Reels & LEDs tool window' is automatically opened too. Set the 'Reel/LED count' to '16', 'Reel/LED spacing' to '5' and click on the previewed 'Rendered LED'.


Now move and resize the score frame to a nearly perfect fit at the left bottom of the backglass.


Add a second score frame, all the settings should be applied from the first LED display, and move the second one to the right.


That's it, nothing more to do. The left score frame is hosting the LEDs 1 to 16, the right one is hosting the LEDs 17 to 32 since the little '1' or '2' in the upper left corner of the score frames is specifying the LED order. With this order the backglass server is able to distribute the LED info from VPinMAME to the LEDs.


Pretty important, save all your changes, don't forget this.
