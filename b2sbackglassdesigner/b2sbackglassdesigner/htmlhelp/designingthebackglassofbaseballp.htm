Designing the backglass of 'Baseball' part one

B2S backglasses for all type of tables ›› An EM or Original 'directB2S' backglass ››

The 'Baseball' backglass project is still open.


We can see three reel displays, for score, homeruns and credits. And a lot bulbs are added as illuminations snippits. This is the backglass in the designer.


Let's start with the bulbs as we can find there some very interesting infos.

Please look at the 'Game over' bulb. To the top left you can see the name of the bulb, 'GameOver'. To the bottom left there is the lamp state, 'Off'. That's not new. To the top right you can see the number '35'. This number is new. Look at the tilt bulb. There you can see the number '33'. And now look at the 'Match' lights, the ten bulbs around the pitcher, from 0 to 9. There you can see the number '34' followed by a '/' and another number from '1' to '10'.

These numbers are the identifiers to access these bulbs from the VP table script. These identifiers are built up of one or two numbers. With these identifiers you can call a command in the VP table script to set the info whether the corresponding bulb(s) should be switched on or off.


Maybe your question right now is: How do I get these numbers? Or: Why is the 'Tilt' bulb the number '33'?


You know this numbers already when you have read the 'Animated backglasses for EM tables' tutorial of Roger Svensson. Roger defined a list of 36 fixed links between an identifier and a backglass counterpart. The only difference is that Roger's list starts with 0 and the designer's list starts with 1. This list is implemented in the designer, no need to memorize it. Since you've chosen the 'B2S data element communication' communication mode three text boxes in the 'Illumination tool window' and two text boxes in the 'Reels & LED tool window' are enabled.


In the 'Illumination tool window' these are 'B2S ID', 'B2S ID type' and 'B2S value'. This is what you have to do for a 'Game over' bulb with corresponding identifier: Open the 'B2S ID type' combo box, move to 'Game Over' ...


... and select it. No need to enter the 'B2S ID' ...


This was pretty simple since normally the 'Game Over' bulb(s) are on or off. No need for more functionality.

It's a bit different when you look at the 'Match' bulbs. Here we have ten bulbs to show the values from 0 to 9. Either all the bulbs are switched off or one of these ten bulbs is on. Now we use the 'B2S value'. Choose 'Match' for the 'B2S ID type' and enter a value larger 0 into the 'B2S value' text box since the value 0 is reserved to switch off all corresponding lights.
