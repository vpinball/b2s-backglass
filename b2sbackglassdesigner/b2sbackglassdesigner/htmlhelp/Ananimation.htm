You have opened the backglass project file of 'Close Encounters of the Third Kind'? If not please do.


To get into the animation configuration tool window please ...

  • ... click on 'Illumination' -> 'Manage animations ...'


Two things are happening right now. First of all the animation configuration window is opened. This is no surprise. Second some illumination frames are visible and the lamp filter is set to '(with name)' since only illumination frames and snippits with a name can be added to an animation.

Please notice: This name doesn't need to be unique. The idea is lamps with the same name act like a group. In an animation you are able to switch a complete group on or off with just one command.


This is how the animation configuration window looks like:


With the combo box and the two buttons at the top of the window  you can start with three basic options:

  • Select an already existing animation to modify it
  • Add a new animation
  • Remove an already existing animation


Open the 'Select animation' combo box and select the third entry, 'Flying Saucer Top (IL1)'. This is what you get:


  • 'Animation name'
    Each animation needs an unique name.
  • 'Interval in ms'
    The base interval between each animation step in milliseconds. Very interesting in combination with the animation steps. Wait to the next chapter for more details.
  • 'Loops'
    The loops the animation is running. 0 means the animation is running endless.
  • 'Join to B2S/ROM ID:'
    A B2S or Rom id the animation is joined to. A B2S id is just a number. A ROM id is composed of one to three letters and a number since a ROM controlled animation is joined to a lamp, a solenoid or a GI string. See more details below at the sample.
  • 'Start animation at backglass startup'
    Check this option to start your animation immediatelly after the backglass startup.
  • 'All lights off at animation start'
    All backglass lights (and I mean all lights except the 'Always on' lights ) are switched off when the animation is started.
  • 'All involved lights off at animation end' or 'All involved lights on at animation end' or 'Reset all lights at animation end' or 'Do not change any lights at animation end'
    Select one of the four options above. At the end of the animation you can switch all involved lights off or on or you can reset all lights to their state at the animation start or you can leave them like they are after the last animation step.
  • 'Run animation steps till end'
    Force the animation to finish all animation steps and loops and not to stop immediatelly when the B2S or ROM id join wants to stop it.
  • 'Lock involved lights during animation'
    Force all in the animation involved illumination bulbs to ignore direct on or off orders from B2S or ROM commands.
  • 'Bring involved lights to front, too'
    This option forces lights and snippits to be brought in front of overlapping lights and snippits.


Right to the 'Animation name' and 'Loops' text box there is a button with a light bulb icon. Click this button to get a preview of your animation at the backglass window in the background.


Well, let's have a look at our sample animation of 'Close Encounters of the Third Kind'.

  • Pretty simple, the animation has got the name 'Flying Saucer Top'.
  • Between each animation step there is a minimum waiting interval of 200 milliseconds. More explanation in the next chapter.
  • The animation is set to run endlessly.
  • The animation is joined to the ROM id 'IL1' since the backglass is ROM controlled. Since the animation is joined to a Rom lamp, solenoid or GI string and the control can be inverted, this id starts with a 'L', a 'S' or a 'GI' or, if inverted, with 'IL', 'IS' or 'IGI' followed by the according number. In our sample the animation is started when the lamp with id '1' is switched off (since inverted) and the animation is stopped when the lamp with id '1' is switched on.
    Since I've done my investigation job at 'Close Encounters' pretty well I know this lamp id is fired when a match is started. This means that this animation is running during the table 'attract mode' and switched off when the first player starts to play.
  • The next two check boxes are not checked so the animation will not start at the backglass start and all the backglass lights stay like they are when the animation starts.
  • 'All involved lights off at animation end' is selected. When the animations is stopped by the fired lamp id all in the animation involved lights are switched off. This is the default setting too.
  • Finally the next two check boxes are not checked so the animation will stop immediatelly when the ROM stops it and the involved lamps are reacting on the direct ROM control.


Got it? It isn't absolutely easy stuff, I know. But there is more because we need some animation steps to tell the animation exactly what to do.
