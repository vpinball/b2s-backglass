Let's start a new backglass for 'The Bally Game Show'.


You are able to start your new B2S backglass project in two different ways:

  • Click on the 'New ...' tool button in the toolbar
  • Click on the menu item 'File' -> 'New ...'.


The following dialog will show up:

  • 'Project name'
    This is the name of your B2S backglass project. This name is for internal use, i.e. the project file and the project subdirectory will be named like this name.
  • 'Backglass (file) name'
    The name of your output file. Name should be like the name of the table.
  • 'Table type'
    Type of the underlying table: Choose between
    • 'EM - Electro Mechanical'
    • 'SS - Solid State Electronic'
    • 'SS DMD - Solid State Electronic with DMD'
    • 'OR - Original'
  • 'Number of players'
    The count of players
  • 'Dual backglass (contains Authentic and Fantasy)'
    A B2S backglass is able to contain two backglass modes: One for a real 'Authentic' look, as close to the original pin as possible, a second for a 'Fantasy' look with some more lightning, some more animations and so on.
  • 'Communication mode'
    Pretty important. Type of communication between table and B2S backglass. Choose between
    • 'VPinMAME ROM communication'
      The output of the designer will be a 'directB2S' file. This file has to be executed by the 'B2S.Server'. All bulbs have to be connected to a ROM output number. This is the new direct ROM controlled B2S style. More infos later.
    • 'B2S data element communication'
      The output of the designer will be a Microsoft Visual Studio 2010 solution. You need this development tool to edit and compile the code and to generate an backglass EXE file. You have to edit the VP table script and add some B2S commands since all scores and bulbs are controlled by these commands. This is the well-known 'Rosve' style. More infos later.
  • 'Destination type'
    Pretty important too: Type of the designer's output. Joined to the 'Destination type'. Possible options are
    • 'B2S.Server 'directB2S' backglass'
      Output of the designer will be a 'directB2S' file. Necessary for communication mode 'VPinMAME'. Available for 'data element communication' too.
    • 'Microsoft Visual Studio 2010 project'
      Output of the designer will be a Microsoft Visual Studio 2010 solution. Not available for communication mode 'VPinMAME'. Available for 'data element communication'.
  • 'Create EM default code (24 reels, 4 rollover, ...)'
    An option for the MS Visual Studio 2010 backglass to create some EM default code in the project files.
  • 'Count of B2S data elements'
    Only available at 'B2S data element communication' communication mode. More infos later.
  • ''B2S DMD' location setting'
    This is NOT the standard VPinMAME DMD, this is a B2S controlled extra DMD. I.e. it can be a simple image at the third monitor or a complex score display like in TAXI. Choose between
    • 'No 'B2S DMD''
      No extra B2S DMD is shown.
    • ''B2S DMD' always on Backglass ('second monitor')'
      (right now I have no idea where to use this option :-) )
    • ''B2S DMD' only on DMD ('third monitor')'
      An extra B2S DMD is always on a third monitor (if existing). Could be i.e. a simple image on the unused third monitor.
    • ''B2S DMD' on Backglass or DMD'
      An extra B2S DMD will be shown on the second monitor in the grill or on the third monitor, like in TAXI.


Please notice:

The red labeled fields are mandatory.
