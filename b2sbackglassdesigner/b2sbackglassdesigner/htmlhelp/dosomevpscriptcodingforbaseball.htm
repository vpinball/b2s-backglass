Let's start this VP script coding from scratch. As I mentioned already what you need from now on is some basic knowledge about 'Visual Pinball' table scripting.


The 'B2S.Server' is enabling the communication between the VP table and the backglass. This is pretty important. And this is our entry into the script coding since we need an instance of this 'B2S.Server'. First of all you have to declare a new variable 'Controller' to the VP script. The VP script code starts normally with statements like 'Option Explicit', 'Randomize' or 'Dim ...'. This is the place where you have to add the following line:


       Dim Controller


And these are the three lines of code we need for a start, very similar to the coding for VPinMAME. Maybe add it in the 'Table_Init' sub:


       Set Controller = CreateObject("B2S.Server")

       Controller.B2SName = "..."

       Controller.Run


The 'Set Controller = ...' command is creating your instance of the 'B2S.Server'. The controller property 'B2SName' is the counterpart to VPinMAME's 'GameName'. DO NOT SET 'GameName' since this property is responsible for the internal calling of VPinMAME in the 'B2S.Server'. Use an unique name for the 'B2SName' property since with this name all backglass settings are saved. Start the backglass with the controller's 'Run' command.


With this controller instance of the 'B2S.Server' you are able to call a bunch of methods to control the backglass: Switch bulbs on or off, show or hide text lights, set reels and credits, and so on. Here is a list of all available commands.


Don't be surprised that some commands are listed twice. You are able to call these commands with different parameter lists.

B2SSetScore

Sets the score of a score display.

A score display is identified by the display id. The display id is visible in the left upper corner of the score frame.

Is working for reels and LEDs. Reels are rolling in the most realistic way.
  • display As Integer: The id of the score display to be changed
  • score As Integer: The new score

B2SSetScorePlayer

Sets the score of the score display for a player. The score display has to be assigned to the 'B2S player no'. When assigning more than one score display to the same player no all these score displays will be connected in-line.

Is working for reels and LEDs. Reels are rolling in the most realistic way.
  • playerno As Integer: The number of the player
  • score As Integer: The new score for a player

B2SSetReel

Sets the value of one digit of the score reels. The reel rolling direction is always up (i.e. changing the value from old value 4 to new value 2 means the reel digit is changed to 5, then to 6, to 7, to 8, to 9, to 0, to 1 and to 2).

Is working only for reels.
  • digit As Integer: The changed digit
  • value As Integer: The new value of the changed digit

B2SSetLED

Sets the value of one digit of the score LEDs.

Is working only for LEDs.
  • digit As Integer: The changed digit
  • value As Integer: The new value of the changed digit

B2SSetCredits

Sets the credits, or to be more precise sets the value of a digit of the credits reels. The reel rolling direction is always direct (i.e. changing the value from old value 4 to new value 2 means the credit reel digit is changed from 4 to 2 without any other changes in between).

Is working only for reels.
  • digit As Integer: The changed digit
  • value As Integer: The new value of the changed digit

B2SSetCredits

Here are some simple samples for Gottlieb's 'Baseball'.

You can see that many ways are leading to the same result. I know this and it's on purpose since you're able to explain your script logic with using the dedicated methods and without writing huge comments.


To switch 'Tilt' on:                'Controller.B2SSetTilt 1' or 'Controller.B2SSetData 33, 1'

To switch 'Tilt' off:                'Controller.B2SSetTilt 0' or 'Controller.B2SSetData 33, 0'

To switch 'GameOver' on:        'Controller.B2SSetGameOver 1' or 'Controller.B2SSetData 35, 1'

To switch 'GameOver' off:        'Controller.B2SSetGameOver 0' or 'Controller.B2SSetData 35, 0'


Maybe a bit more interesting at the match lights, number '34'. There are ten bulbs with this identifier since ten bulbs do have a similar meaning.

To switch 'Match 5' on:        'Controller.B2SSetMatch 5' or 'Controller.B2SSetData 34, 5'

To switch 'Match 9' on:        'Controller.B2SSetMatch 9' or 'Controller.B2SSetData 34, 9'

To switch 'Match 0' on:        'Controller.B2SSetMatch 10' or 'Controller.B2SSetData 34, 10'

To switch all 'Match' bulbs off:        'Controller.B2SSetMatch 0' or 'Controller.B2SSetData 34, 0'


Now we wanna set the value of the score display to a value of 350 points. It's made of four digits, starting at 'B2S start digit' 1, is assigned to 'player 1' and has the display id '1'.

The first recommended way to set the score display is using the player assignment:

'Controller.B2SSetScorePlayer1 350'

The second recommended way to set the score display is using the display id:

'Controller.B2SSetScore 1, 350'

Ant the third but not recommended way is to set each digit separately:

'Controller.B2SSetReel 1, 0

Controller.B2SSetReel 2, 3

Controller.B2SSetReel 3, 5

Controller.B2SSetReel 4, 0'


Now we wanna set the value of the homerun display to 23 homeruns. It's made of two digits, starting at 'B2S start digit' 7, is not assigned to any player no and has the display id '2'.

No way to use the first recommended way because there is no player assignment done


The second recommended way to set the score display is using the display id:

'Controller.B2SSetScore 2, 23'

Ant the third but not recommended way is to set each digit separately:

'Controller.B2SSetReel 7, 2

Controller.B2SSetReel 8, 3



And here are some VP table script code samples to explain the commands. I'm still at Gottlieb's 'Baseball':

Used B2S command

Sample code for 'Baseball'

Description

(startup)

Set Controller = CreateObject("B2S.Server")

Controller.B2SName = "Baseball"

Controller.Run

Creates an instance of the  'B2S.Server', available as 'Controller' in the script. Sets the 'B2SName' to a unique name 'Baseball' and starts the backglass with 'Run'.

B2SSetTilt

Sub B2SSetTilt(value)


   Controller.B2SSetTilt value


End Sub

Pretty simple, the visibility of id '33' is set to 'value'.

Since there is one bulb for 'Tilt' at 'Baseball' and this bulb is entered without a 'B2S value' in the designer 'value' '0' switches the 'Tilt' light off and any other 'value' switches it on.

B2SSetScoreRolloverPlayer1

Sub B2SScore10K()


       Score10K = Score10K + 1

       

       If Score10K <= 4 Then

               Controller.B2SSetScoreRolloverPlayer1 Score10K

       Else

               Controller.B2SSetScoreRolloverPlayer1 5

       End If


End Sub

At 'Baseball' we have 5 score rollover lights, all for player 1 (since 'Baseball' is a one player pin). The lights show the values '10000', '20000', '30000', '40000' and '50000'.

The sub 'B2SScore10K' is called every time the score is running over 10000 points. The 'Score10K' variable stores how many times a rollover happens.

At the rollover from 10000 to 10001 points 'Score10K' is incremented to 1 and the 'B2S.Server' is told to show the light joined to id '25' and value '1' (since the id of 'B2SSetScoreRolloverPlayer1' is the recommendation of '25').

At the next rollover from 20000 to 20001 points 'Score10K' is incremented to 2 and the 'B2S.Server' is told to show the light joined to id '25' and value '2'.

And so on.

All above 50000 will be visualized with the '50000' light joined to value '5'.

B2SSetCredits

Sub B2SSetCredits(credits)


       Controller.B2SSetCredits credits


End Sub

Sets the credit reel to the value of 'credits'. Since I'm using 'Controller.B2SCredits' with just one parameter the credit reel digit is the recommended '29'.

B2SSetScorePlayer1

Sub B2SSetScore(value)


       Controller.B2SSetScorePlayer1 value

               

End Sub

The current score of player 1 is set to 'value'.

B2SSetScore

Sub B2SAddRun()


       b2sruns = b2sruns + 1

       Controller.B2SSetScore 2, b2sruns

       

End Sub

This sub is called every time a homerun is counted. This count of homeruns is stored in the variable 'b2sruns'.

This value of 'b2sruns' is sent to the backglass reel display with id '2'.


In the case of 'Baseball' and since the homeruns' reel display starts at 'B2S start digit' 7 this command would be exact the same:

Controller.B2SSetScorePlayer2 b2sruns
